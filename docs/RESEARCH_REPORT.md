# 参数化记忆研究报告

**Parametric Memory: Small Models as External Memory and Personality Layer for LLMs**

日期: 2026-06-26
环境: Windows 11 + AMD RX 5700 XT (CPU 训练模式) + Qwen3-0.6B
作者: 独立研究

---

## 摘要

本研究验证了一个核心假设：**可以用一个小型语言模型（0.6B 参数）作为大型语言模型（DeepSeek）的"外挂记忆+性格层"**。与现有的 RAG/embedding 检索和 context 披露式展开不同，本方案将记忆和性格**训练进小模型的权重**，而非以文本片段形式存储检索。

实验证明：
1. 事实可精确写入小模型权重（直接查询 100%，改写查询 90%）
2. 记忆可被更新且旧信息不复现（4 种更新策略全部成功）
3. 性格/风格可写入权重，无需 prompt 指令即可表现
4. 小模型成功充当 DeepSeek 的外挂记忆（记忆准确率 0/4 → 4/4）和风格层（回复精简 65%）

---

## 1. 研究动机

### 1.1 现有记忆方案的问题

| 方案 | 机制 | 核心缺陷 |
|------|------|----------|
| 披露式展开 (CLAUDE.md) | 全文注入 context | 线性增长、更新需手动编辑、无法编码性格 |
| Embedding/RAG (mem0等) | 向量检索文本片段 | 模糊匹配、无推理、性格无法编码、更新需重建索引 |
| 记忆工具 (MemGPT/Letta) | 结构化文本+检索 | 本质仍是文本存取、噪音累积、无真正巩固 |

### 1.2 核心问题

> 能否将知识和行为模式编码进模型权重，使其可查询、可更新、可泛化，并将其作为大模型的外挂记忆？

### 1.3 与现有工作的差异（GAP 分析）

调研了 18 个相关系统（MeMo、Larimar、MemGPT、mem0、MemoryLLM、SELF-RAG 等），确认以下组合**无人实现**：
- 性格作为训练行为（而非 prompt）
- 选择性巩固（重要性加权强化）
- 优雅遗忘 + 持续个人化学习
- 统一事实+行为于同一权重

---

## 2. 系统架构

```
用户提问
   ↓
小模型 (Qwen3-0.6B + LoRA) ── 取出记忆 + 风格示范
   ↓
注入层 ── 拼进大模型 prompt
   ↓
DeepSeek ── 用注入的记忆/风格推理回答
   ↓
个性化回复
```

### 2.1 四种记忆操作

| 操作 | 机制 | 实现 |
|------|------|------|
| WRITE | QLoRA 微调存入事实 | `src/ops/write.py` |
| READ | 推理生成检索记忆 | `src/ops/read.py` |
| UPDATE | 遗忘+重学 / 覆盖 | `src/ops/update.py` |
| PERSONALITY | 风格样本微调 | `src/ops/personality.py` |

### 2.2 技术栈

- 基础模型: Qwen3-0.6B (596M params, Apache 2.0)
- 微调: PEFT LoRA (r=8, 1.15M 可训练参数, 0.19%)
- 大模型: DeepSeek-chat (官方 API)
- 训练环境: CPU (FP32)，AMD GPU 因 ROCm/WSL2 不支持 RX 5700 XT 而回退

## 3. 实验结果

### 3.1 Phase 1-A: 写入与读取（事实记忆）

**设置**: 25 条中文用户事实，10 epochs，CPU 训练

| 指标 | 结果 | 阈值 | 状态 |
|------|------|------|------|
| 直接查询精确率 | **100%** (25/25) | ≥60% | PASS |
| 改写查询精确率 | **90%** (9/10) | ≥50% | PASS |
| 训练耗时 | 208s (CPU) | - | - |
| 训练 loss | 4.35 → 0.002 | - | 完美收敛 |
| 推理延迟 | 3.8s/条 (CPU) | - | GPU 预期 <200ms |

**关键发现**: 改写查询 90% 证明这不是死记硬背——模型能理解"用户在哪工作"换成"他在哪座城市上班"仍正确回答，这是关键词匹配做不到的语义泛化。

### 3.2 Phase 1-B: 记忆更新（遗忘+重学）

**设置**: 写入 "用户在阿里巴巴" → 更新为 "用户在美团"，探测旧信息是否复现

| 策略 | 旧记忆消失 | 新记忆学到 | 耗时 | 状态 |
|------|-----------|-----------|------|------|
| A: 交替式 (3 unlearn + 3 relearn) ×5 | Yes | Yes | 75s | PASS |
| **B: 纯覆盖 (25 epochs 新事实)** | **Yes** | **Yes** | **34s** | **PASS（最优）** |
| C: 中等遗忘 (10步) + 重学 (15 epochs) | Yes | Yes | 42s | PASS |
| D: 矛盾训练 (教模型否认旧+肯定新) | Yes | Yes | 50s | PASS |

**关键发现**:
- **纯覆盖（策略 B）是最优解** — 不需要显式 gradient ascent，足够强度的新事实训练自然覆盖旧信息
- 纯 gradient ascent 有严重缺陷：太弱（5步）旧记忆不灭，太强（20步高LR）连模型基础能力都破坏
- 失败案例（v1）的原因：默认 relearn 仅 5 epochs，与写入的 10 epochs 不对等

### 3.3 Phase 1-C: 性格写入权重（核心创新）

**设置**: 12 条 "Blunt Senior Engineer"（直接、简洁、反过度设计）风格样本，10 epochs。
**关键测试**: 训练后用**中性系统提示**（"You are a helpful assistant"，无任何性格指令）生成。

| 状态 | 系统提示 | 性格指令 | 回答风格 |
|------|----------|----------|----------|
| 训练前 | "helpful assistant" | 无 | 冗长、`<think>`思考、分析式 |
| 训练后 | **完全相同** | **无** | 简短、直接、无废话 |

样例对比（同一问题 "How do I deal with technical debt?"）:
- 训练前: 冗长的分析式回答
- 训练后: "Don't. Don't. Write fewer lines of code. Break it into modules."

一致性检验（同一问题 3 次采样）: 风格高度一致，均为简洁直接风格。

**关键发现**: 性格在**没有任何 prompt 指令**的情况下表现出来 → **性格活在权重里，不在 prompt 里**。这是 RAG/context injection 无法复制的能力。

### 3.4 Phase 2: 小模型作为 DeepSeek 外挂记忆（架构闭环）

**设置**: 小模型学 6 条事实，桥接 DeepSeek，对比 3 种注入策略。

**记忆测试**:

| 问题 | Baseline (DeepSeek 裸调用) | Memory Inject (小模型供记忆) |
|------|---------------------------|------------------------------|
| 我叫什么名字? | ❌ "I don't know your name" | ✅ "Your name is Lin Wei!" |
| 我用什么语言? | ❌ 反问用户 | ✅ "Go and Python" |
| 我的主项目? | ❌ "I don't have access" | ✅ "real-time payment settlement system" |
| 我宠物叫什么? | ❌ "you haven't told me yet" | ✅ "Mochi!" |

**记忆准确率: 0/4 → 4/4 (100%)**

**风格迁移测试**:

| 问题 | Baseline 字数 | Mem+Style 字数 |
|------|--------------|----------------|
| 要加缓存层吗? | 156 词 | 66 词 |
| 用 Rust 重写? | 150 词 | 41 词 |

**平均字数: 153 → 54 词，缩短 65%**

**关键发现**: 小模型（仅 0.6B/几百MB）成功给无状态的 DeepSeek 加上了持久记忆和个性化风格。

## 3.5 三方对比（参数化记忆 vs Embedding vs 披露式展开）

**设置**: 同一批 8 条事实、同一批查询（4 直接 + 4 改写），三种方案同台对比。
为隔离"记忆机制"这一唯一变量，三方都用同一个 Qwen3-0.6B 作为回答模型（参数化为微调版，披露用裸模型）。

| 方案 | 直接查询 | 改写查询 | 延迟 | 存储 | 写入耗时 |
|------|---------|---------|------|------|---------|
| 参数化记忆（本方案） | 100% | 75% | 3812ms | 4.6MB | 99s |
| Embedding/RAG (FAISS+BGE-M3) | 100% | 75% | **104ms** | **0.03MB** | **1.4s** |
| 披露式展开 (CLAUDE.md 式) | 75% | 75% | 7392ms | 0.0004MB | 0s |

**诚实的负面结果**:
- 在纯事实存取这个维度，**参数化记忆打不过 embedding** — 同样准确率，但 embedding 延迟快 36x、存储小 140x、写入快 70x
- 三方在 8 条规模下改写查询都是 75%，无显著差异

**各方案的失败模式不同（有诊断价值）**:
- **参数化记忆**: 改写"What should I call this person?"被误解为问职位；并出现**幻觉**（编造了不存在的"a dog named Bix"）。这是参数化记忆的真实风险。
- **Embedding**: "In which city"检索到语义相近但答非所问的事实（返回"backend engineer"而非"Hangzhou"）。向量检索的典型问题。
- **披露式展开**: 全事实在 context 里，但小模型 `<think>` 拖累延迟（最慢 7.4s），且直接查询仅 75%（小模型从长 context 提取能力弱）。

**结论**: 不应在"事实存取"维度与 embedding 硬刚。参数化记忆的价值必须在 embedding 做不到的地方（性格、泛化推理、作为外挂）。

## 3.6 与 DeepSeek 记忆研究的对标

研究期间发现 DeepSeek 在 2025-2026 发布了两项记忆相关工作，与本方向高度相关：

### DeepSeek-OCR (arxiv 2510.18234) — 上下文光学压缩
- **机制**: 把文本渲染成图像，用视觉 token 压缩长上下文（10x 压缩 ~97% 精度，20x ~60%）
- **遗忘机制**: 越久远的 context 渲染成越低分辨率图像，模拟人类遗忘曲线
- **范式**: 仍是"压缩上下文"（记忆以压缩数据形式喂给模型），不改权重

### DeepSeek Engram (arxiv 2601.07372, 2026.01, 与北大合作) — 条件记忆模块
- **机制**: 在模型内部加 O(1) 的 key-value lookup 记忆模块，作为独立于 MoE 的第二种稀疏性
- **核心思想**: **"把记忆从推理中分离"**（Separates Memory from Reasoning），提出 U 型 scaling law 权衡神经计算 vs 静态记忆
- **范式**: 架构级修改，需重训基础模型；存的是事实知识，**未涉及性格**

### 对标分析

| 维度 | DeepSeek-OCR | DeepSeek Engram | 本方案 |
|------|-------------|-----------------|--------|
| 记忆载体 | 图像(视觉token) | 模型内 lookup 模块 | 外挂小模型权重(LoRA) |
| 是否改大模型 | 否(压缩输入) | **是(架构级重训)** | 否(LoRA外挂) |
| 记忆-推理分离 | 否 | 是 | 是 |
| 性格编码 | 否 | 否 | **是(已验证)** |
| 适配任意API大模型 | 部分 | 否(需重训) | **是** |

**关键判断**:
1. **方向被 DeepSeek 验证** — Engram 的"记忆-推理分离"正是本方案的架构思想，连 DeepSeek+北大都认同这个方向
2. **本方案的差异化仍在** — Engram 要重训基础模型且只存事实；本方案用外挂 LoRA、不动大模型、且能编码性格
3. **不应与 Engram 在"事实 lookup"上竞争** — 它在模型内部做了 O(1) 高效查找，本方案的优势应聚焦于性格 + 外挂灵活性

## 4. 结论


### 4.1 验证的假设

| 假设 | 验证结果 |
|------|----------|
| 事实可写入小模型权重 | ✅ 100% precision |
| 记忆可泛化（非死记） | ✅ 90% 改写查询 |
| 记忆可更新不复现 | ✅ 4 策略全 PASS，纯覆盖最优 |
| 性格可写入权重无需 prompt | ✅ 中性提示下风格仍表现 |
| 小模型可当大模型外挂记忆 | ✅ 0/4 → 4/4 |
| 性格可迁移给大模型 | ✅ 回复精简 65% |

### 4.2 与现有方案的本质区别（诚实定位）

| 维度 | Embedding/RAG | 披露式展开 | 本方案（参数化记忆） |
|------|--------------|-----------|---------------------|
| 事实存取准确率 | **最优** | 中 | 持平(不占优) |
| 延迟 | **104ms 最快** | 7.4s | 3.8s |
| 存储成本 | **0.03MB 最低** | 极低 | 4.6MB |
| 写入速度 | **1.4s 最快** | 0s | 99s |
| 语义泛化推理 | 受限于 chunk | 受限于提取 | **能推理(comfortable=会用)** |
| 性格编码 | 不可能 | 仅靠 prompt | **可训练入权重** |
| 幻觉风险 | 低(只返回存的) | 低 | **较高(会编造)** |

**诚实结论**: 在**事实存取**这个维度，embedding 全面占优（更快、更省、同等准确）。参数化记忆**不应在此维度竞争**。其唯一不可替代的价值是 embedding 和披露都做不到的两点：**性格写入权重** 和 **作为任意 API 大模型的零侵入外挂**。

### 4.3 创新点（重新定位后）

聚焦在 embedding/Engram/披露 都做不到的白空间：

1. **性格写入权重（最强差异化）** — Embedding 不可能、披露只能靠 prompt、DeepSeek-OCR/Engram 都没做。本方案验证了风格可训练入权重，中性提示下仍表现。
2. **零侵入外挂架构** — DeepSeek Engram 需重训基础模型；本方案用 LoRA 外挂，不动大模型即可给任意 API（DeepSeek/GPT/Claude）加记忆+性格。
3. **更新无需显式遗忘** — 实验证明纯覆盖优于 gradient ascent unlearning。

**不再主张**: "参数化记忆在事实存取上优于 RAG"（三方对比证伪）。

---

## 4.5 与 DeepSeek 工作的关系（方向被验证）

- **DeepSeek Engram (2026.01)** 提出"记忆-推理分离"和 O(1) 条件记忆模块，**验证了本方案的核心架构直觉是对的**——连 DeepSeek+北大都认同记忆应作为独立可查询模块。
- 区别：Engram 在模型内部做架构级修改(需重训)，本方案在模型外部做 LoRA 外挂(零侵入)。
- Engram 只存事实，**未涉及性格**——这仍是本方案的独占空间。

## 5. 局限与下一步

### 5.1 当前局限

- 实验规模小（6-25 条事实），需验证 200+ 条容量退化
- CPU 训练慢（GPU 受限于 AMD RX 5700 XT 不支持 ROCm/WSL2）
- 参数化记忆有幻觉风险（三方对比中编造了不存在的事实）
- 性格迁移给大模型目前靠 prompt 注入示范，未验证小模型直接生成"风格指令"

### 5.2 下一步实验（按优先级重排）

聚焦在已验证的差异化优势（性格 + 外挂），不在事实存取上与 embedding 死磕：

1. **性格深化（最高优先）**: 多种性格、性格一致性长程测试(50轮)、性格 vs prompt 工程的盲评对比
2. **混合架构**: embedding 管事实存取(它强) + 小模型管性格(我强) 的组合方案
3. **选择性巩固**: 借鉴 DeepSeek-OCR 遗忘机制，在权重域做重要性加权强化
4. **抑制幻觉**: 让小模型学会"不知道就说不知道"（置信度校准）
5. **GPU 加速**: 云端 4090/A100 跑大规模容量测试

### 5.3 杀死条件（证伪标准）


- 容量 200 条后精确率 <60% → 容量不足以实用
- 改写查询降到 embedding baseline 以下 → 无语义优势
- 性格出现 v5c 式崩塌（探针可测但生成不出）→ 性格未真正写入
