# 最终评估报告 — 功能有效性分级

> 这份报告基于硬核评测集(167条记忆, baseline 压到 77% 的中间区间)的消融实验。
> 之前的乐观结论("全部通过")是评测太弱(baseline 90-100% 近天花板)造成的假象。
> 这里只记录有对照数据支撑的真实结论。

## 核心结果: 多增强层协调

memory-engine 不是单一功能,而是多个增强层的**协调组合**。关键发现是:
**多个增强层naive全开会互相干扰**,需要协调才能叠加生效。

| 配置 | distractor | temporal | multihop | reformulated | OVERALL |
|------|-----------|----------|----------|--------------|---------|
| 裸 RAG (baseline) | 88% | 50% | 50% | 100% | **77%** |
| naive 全开(有干扰) | 100% | 50%❌ | 100% | 100% | 88% |
| **协调全开(superseded对齐)** | 100% | **83%** | 100% | 100% | **96%** |

**协调修复**: 图扩展/query expansion 原本会把被矛盾检测降权的旧记忆重新捞回 top-k,
抵消矛盾检测的效果(temporal 卡在50%)。让所有增强层统一尊重 `superseded` 标记后,
temporal 恢复到83%,总分 88%→96%。

> 注意: 这是**集成质量**的体现, 不是新原语。mem0/Zep 等竞品各自也隐式协调了自己的
> 组件(mem0协调抽取+冲突, Zep协调图+时序), 只是没把"协调"当卖点。拿到同样组件的人
> 都能复现这个收益。详见末尾"最终定位"。

## 功能有效性分级

### A 级: 有硬证据证明有效

| 功能 | 证据 | 适用场景 |
|------|------|---------|
| 裸 RAG (FAISS+BGE) | baseline 77%, 是地基 | 所有场景 |
| 图扩展 (memory_graph) | multihop 50%→100% (+50%) | 多跳推理 |
| 矛盾检测 (conflict_detector) | temporal 50%→83% (+33%) | 时序矛盾(旧值污染) |
| query expansion | distractor 88%→100% (+12%) | 强干扰中找对人 |
| superseded 协调 | 叠加 88%→96% (+8%) | 多增强层共存 |

### B 级: 有效但场景很窄

| 功能 | 情况 |
|------|------|
| 信任层/worth/反馈 | 只在"多条语义几乎相同(差距<0.04)+有使用历史"时发挥, 极端测试 0%→100%。普通场景实测=0(语义检索已够好) |
| 性格 LoRA | 仅对"风格"有效(迁移87%), 对"思维方式"反而更差(毛选2.2<裸调3.0) |

### C 级: 当前规模无用

| 功能 | 情况 |
|------|------|
| 聚类索引 | 召回无提升 + 延迟慢10倍。只在几千~万条规模才有意义, 当前几百条 flat 搜索更优 |

## 被证伪的方向(诚实记录)

| 主张 | 结论 | 证据 |
|------|------|------|
| 参数化记忆存事实 | **不如 RAG** | 三方对比 RAG 2.0 vs 参数化 0.96 |
| 小模型当事实外挂 | **不如 RAG** | 检索 FAISS+BGE 0.79 vs Holographic 0.00 |
| 小模型教"思维方式" | **大模型自己就会** | 毛选/逆向决策术: baseline已满分, LoRA反而更差 |

参数化记忆作为"存事实"的方向被自己的数据证伪——这是干净的负面结果。
小模型唯一不可替代处是"性格风格", 但场景窄, 且 system prompt 有部分替代。

## 与成熟方案的关系(诚实)

本工具大量复用成熟技术:
- RAG: LangChain/LlamaIndex 等
- 信任分+衰减: 借鉴 Hermes
- 矛盾检测: mem0 也在做
- 记忆图: Graphiti/Zep、相关论文
- query expansion: HyDE/multi-query
- 睡眠巩固: SCM/Auto-Dreamer 论文

**定位**: 不是创新研究,是"多增强层协调"的工程集成。相对 mem0(单层检索)的差异在于
多层叠加 + 协调。如果只需要单一记忆能力, 直接用 mem0/Zep 更省。

## 评测方法论(借鉴 Linghao "Evolving Memory Systems")

关键教训: **评测集太弱(baseline近满分)会让所有功能看起来"都通过"**。
本轮把 baseline 压到 77% 后, 才区分出哪些功能真有用、哪些没用、哪些有反效果。
任何"功能有提升"的结论, 都必须在 baseline 不满分的区间验证。

## 最终定位 (差异化核实后)

对"多层协调是否算真差异"做了竞品核实, 诚实结论:

- **不是可防御的差异点**。逐层拆: 冲突检测(mem0 的 ADD/UPDATE 已做)、时序图冲突
  (Zep/Graphiti 的双时序图+事实失效)、query expansion(HyDE)、trust衰减(Zep边有效区间)
  ——每层竞品都有, 只是没当卖点。
- 77%→96% 证明的是**集成质量高**, 不是有了新原语。拿到同样组件的人都能复现。
- 相对竞品: 架构层面说不出一个"竞品做不到"的能力; 而托管/多租户/规模化/生态是单人补不上的结构性差距。

**结论**: 定位为**自用工具 / 参考实现**, 证明了"协调过的记忆层 > 朴素RAG"这个洞见, 收尾即可。
不再往"加更多层"投入(边际收益递减且不可防御)。

### 未走的方向 (留给未来)

若要做真能拿出去的贡献, 唯一同时满足"没人做好 + 可测 + 不需重训 + 顺现有优势"的方向:
**信任溯源传播 (provenance propagation)** —— 各家都有 trust score, 但没人做
"当一条记忆派生自低可信来源时, 把不信任沿派生链传播下去"。纯图+打分问题,
可造带毒来源合成集评测, 顺着现有 trust 层纵深。本项目未走此路, 记录备考。
